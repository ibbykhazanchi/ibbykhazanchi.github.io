Command Line Two-Player Chess

A command line chess game for two people

Calling out all Queen's Gambit fans!! This is a program I had to write for my Software Methodology class. It's exactly what it sounds & looks like: a two-player chess game played on the command line with built-in game control. This program was written in Java.

On the surface, the program seems pretty simple, but in practice, it came out to be quite difficult. We used a bunch of OOP principles to make our life easier, including the use of abstract classes and interfaces for polymorphism. For example, we made one of our abstract classes was ChessPiece, and all of the individual piece classes would extend this class. We also made interfaces for diagonal movers (Queen, Bishop) as well as forward movers (Queen, Rook, Pawn) that included a bunch of default methods.

I really liked this project because I saw how OOP can allow for code reuse, making a developer's life easier. The hardest part was figuring out an algorithm to detect CHECKMATE. What I did was check all of the king's vulnerabilities (what pieces could potentially put the king in check) and from there, determine if a checkmate could be made. I thought about how I would determine a check-mate in real life and then thought about how I could translate that into code in the most algorithmically efficient way.The end result is a highly intelligent chess CPU that knows what's a legal move and what's a checkmate. Future projects might be making an AI for the player to play against.
